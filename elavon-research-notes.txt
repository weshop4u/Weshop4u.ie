Elavon Converge Integration Research Notes
==========================================

Elavon Converge (formerly Virtual Merchant) - Key Details:

Required Credentials:
1. Account ID (ssl_account_id) - Also known as Merchant ID
2. User ID (ssl_user_id) - Must be set as API user with session token permission
3. PIN (ssl_pin) - Generated within Virtual Merchant admin page

Integration Methods Available:
1. Checkout.js - JavaScript library that collects card data in browser, sends directly to Converge
   - PCI SAQ A-EP compliance level
   - Best for web apps
   - Session token based flow
   
2. Hosted Payment Page (HPP) - Low-code, redirect to Elavon's page
   - PCI SAQ A compliance (lowest burden)
   - OAuth 2.0 authentication
   - Best for simplicity and security
   
3. XML API - Direct API integration
   - Most flexible
   - Higher PCI compliance requirements

Supported Transaction Types:
- Authorize (ssl_transaction_type = ccauthonly)
- Capture (ssl_transaction_type = ccforce)
- Purchase/Sale (ssl_transaction_type = ccsale)
- Refund (ssl_transaction_type = ccreturn)
- Void (ssl_transaction_type = ccvoid)
- Recurring payment

Converge API Endpoints:
- Production: https://api.convergepay.com/VirtualMerchant/process.do
- Demo/Sandbox: https://api.demo.convergepay.com/VirtualMerchant/process.do

Session Token Flow (Checkout.js):
1. Server requests session token from Converge (POST with merchant_id, user_id, pin, amount, transaction_type)
2. Server receives session token
3. Client uses session token to open Checkout.js payment form
4. Customer enters card details in Checkout.js form (data goes directly to Converge)
5. Converge processes payment and returns result
6. Client receives callback with transaction result

For Mobile (React Native):
- Checkout.js works in WebView
- HPP works via redirect (Linking.openURL or WebView)
- Recommended: HPP for mobile app (simplest, most secure)
- Recommended: Checkout.js for website (better UX, no redirect)

Currency: EUR (for Ireland)
